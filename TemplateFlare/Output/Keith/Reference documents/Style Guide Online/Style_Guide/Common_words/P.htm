P

path name (n), path-name (adj)

Two words (noun); hyphenated (adjective).

perform | carry out | run

For commands, use perform, not carry out. For applications, use run. For user actions, use perform. Avoid using execute at all times.

Yes: Run the Scanner application.

Yes: Perform cartridge replacements as soon as…

No:  Carry out cartridge replacements as soon as…

No:  Execute the Scanner application.

peripheral

Refers to printers, bar code readers, disk drives, keyboard, and so on. For user documentation only, use accessories.

pop-up

Use context menu in user documentation. Pop-up window is acceptable in references to context-sensitive Help.

power down, power up, power-cycle

Use these terms when no switch is available or connection to power supply.

press

Use press, not depress or type, when pressing a key initiates an action within an application.

Use use in situations when press might be confusing.

Do not use strike or hit.

► See "Banned words"
printout (n)

One word.

procedure

In documentation, a procedure is a short description of the steps a user takes to complete a specific task. In printed and online documentation, procedures are set off from the main text by their formatting.

program

Do not use program; use application.

► See "Banned words"
prompt (v)

Do not use prompt as a noun to mean 'message'. Use prompt as a verb to mean the system is requesting information or an action from the user.

Use prompt as a noun only when used as a proper noun.

Yes: Command line prompt

Yes: Voice prompt

Yes: The system will prompt you...

No:  A warning prompt is shown.

push button (n), push-button (adj)

Avoid in all contexts; instead, use button or refer to the button by its name alone.
