Topic construction

A topic is a single piece of content that covers one subject. This section describes how to construct a topic, the content of a topic and the different types of topics.

There are three main types of topics. These definitions follow the DITA standards:

Concept - Answers the question "What is this about?".

Task - Answers the question "How do I ...?".

This topic must contain a procedure or task description. Each topic must only cover one main task or procedure.

Reference - Contains lists and specifications required to support task or concepts.

Each topic should be self-contained and not cover multiple subjects. Preferably they should only cover one piece of subject matter.

Topics should not be too long (a page in length).

Write a topic with single-sourcing in mind. Alter sentences slightly so that they become generic and can be used in multiple locations.

If required, use conditional text. However, this must be kept to a minimum.

For more information, See "Conditional text" on page 1

Do not have empty topics; topics with a heading only, unless absolutely necessary for document construction.
