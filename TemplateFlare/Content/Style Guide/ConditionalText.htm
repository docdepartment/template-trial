Conditional text tags

Apply conditional text tags to content to enable different text to be shown or hidden in different outputs. This could be alternative content, or a description of a feature which only applies to one type of product.

Apply conditional text to whole paragraphs, rather than in-line.

Do not nest conditional tags.

Keep use of conditional text tags to a minimum.

To avoid conditional tag sets becoming overly complex, apply these rules:

When writing content, be aware of the output types that the content is to be used for.

Conditions should be positive and inclusive.

Do not create NOT conditions.

One primary dimension of conditions should be established per project.

For example, this dimension could be Guide Type (User Guide / Quick-Start Guide). This dimension can contain as many conditions as required.

No more than two other dimensions should be created.

These two should only contain three conditions each. These could be, for example, Product Family and Output Type.
