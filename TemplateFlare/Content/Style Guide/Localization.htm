Localization

All documentation must be written with localization in mind. Localization includes translating words, graphics, and the format of the text (for example, right-to-left languages).

Use a clear and plain writing style.

Avoid splitting phrasal verbs.

Switch off the Integrated Control Keyboard

Switch the Integrated Control Keyboard off

Make sure that pronouns and their referents are explicit.

Clarify any ambiguous phrases.

For example, English words ending in '-ing' can act both as verbs, as in 'meeting the requirements', or nouns, as in 'requirements for the meeting'. 'Meeting requirements' can be an ambiguous phrase.

Include optional pronouns to make verb structure more explicit

Select the settings that you need

Select the settings you need

Avoid using the passive- this can be complicated to translate.

Content structure must be easy to follow.

Avoid hanging sentences, for example when introducing lists.

See "Lists"

Do not refer to seasons, weather, time zones, or holidays.

Use common phrases when appropriate.

See "Common phrases"

Do not refer to left or right parts of a screen.

These may be reversed for different languages.
